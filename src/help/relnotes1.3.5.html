VisIt 1.3.5 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.3.5

  • VisIt now supports for connecting to running simulations that have been instrumented with the new VisIt/Simulation library. When VisIt connects to a running simulation, the simulation in essence becomes a VisIt compute engine and serves data from the simulation back to VisIt where it can be plotted.
  • VisIt has a new window called Interactors that lets you set the options for VisIt's zoom interactor. You can open the Interactors window by clicking on the Interactors... option in the Options menu in VisIt's Main window. Zoom mode has been altered so it shows guidelines by default. If you prefer the previous behavior, set the options for zoom mode in the Interactors window.
  • VisIt can now pick on all variables at once if you put all in the secondary variables in the Pick window
  • The error messages that VisIt issues when it cannot open a file have been improved so it is more apparent what is happening.
  • The Z-Axis of the Surface plot now uses the plotted variable's name.
  • VisIt's rubber band zoom mode has been changed so holding down the Control key while drawing the zoom rectangle causes VisIt to zoom out.
  • VisIt now displays ijk coordinates in terms of AMR-level for SAMRAI databases.
  • VisIt now uses ghost nodes, when appropriate, to speed up operations at domain boundaries.
  • VisIt's Query list is now sorted when it appears in the Query window.
  • More of VisIt's wide variety of database reader plugins are available on all platforms.
  • Changing the size of a point mesh no longer causes the plot of the point mesh to be recalculated on the compute engine.
  • Queries can now be run on the Histogram plot so you can, for example, query the area under the Histogram plot's curve.
  • Mesh plots are no longer automatically drawn in opaque mode if a point mesh is also plotted in the vis window.
  • The IndexSelect operator attributes window has been changed to make it easier to increment or decrement the min/max ranges.
  • The Num Zones query has been changed so it reports information for the original data rather than reporting data for only data visible in the vis window.
  • VisIt now has preliminary support for global zone/node numbers for databases with multiple domains.
  • VisIt's Contour plot now reports progress as it is being generated on the compute engine.
  • The OnionPeel window's Category and Set selection controls are now disabled when there is onlt 1 choice.
  •  
  • Queries now work on point meshes.
  • The Reflect operator now creates ghost nodes to get rid of internal geometry.
  • VisIt has a new database reader plugin that can read data from Pixie3D files.

Bugs fixed in version 1.3.5

  • VisIt's Python interface crashes immediately on sc.
  • VisIt's compute engine sometimes crashes after reopening a file when scalable rendering is in Auto mode.
  • The compute engine crashes after doing a lineout.
  • Faces are drawn inside-out for some databases.
  • VisIt's compute engine crashes when processing a Contour plot of 2D data in 3D space.
  • VisIt does not correctly copy cell-centered data for 2D data in 3D space.
  • VisIt's Flash and Z database reader plugin never freed certain data arrays, causing the compute engine to bloat and eventually crash while generating frames for a movie.
  • VisIt's compute engine crashes when processing a Subset plot for a KullLite database.
  • Transparent geometry is not cleared out of the vis window when switching from 3D transparent contours to a 2D contour variable.
  • Picking on VisIt's Boundary plot does not output the same results as MeshTV.
  • The Histogram plot takes materials even though that functionality is not yet implemented. The plot has been altered so it only takes scalar variables as input.
  • Automatic file grouping does not work properly with multiple databases that have similar base names.
  • Some of VisIt's parallel libraries have duplicate symbols.
  • The Mesh line offset is not large enough when rendering images offscreen.
  • VisIt's camera source code is out of date with VTK.
  • VisIt's structured domain nesting code, used with AMR databases, only checks immediate children in the AMR hierarchy.
  • VisIt's SAMRAI database reader plugin does not advertise cycles and times as being accurate.
  • The IndexSelect operator does not handle module problems.
  • VisIt's SAMRAI database reader does not populate certain I/O information, which helps VisIt assign domains to processors in parallel.
  • VisIt needs to better handle ghost zones in the database and visualization pipeline.
  • Vector and Tensor plots do not correctly report triangle counts to change when VisIt switches into scalable rendering mode, which caused the X-server to crash due to too much geometry being sent to the viewer.
  • The magnitudes of vectors plotted by the Vector plot are not correct after slicing.
  • Reset view only works when there are plots in the vis window.
  • VisIt's legend turned on unexpectedly.
  • The available color tables did not appear in the Pseudocolor plot attributes window.
  • Node pick of a sliced FilledBoundary plot appears to return points that are not real nodes in the problem.
  • VisIt's Choose center function does not use all plots.
  • The Mesh plot's test for automatically determining opaqueness does not take into account whether other plots in the plot list have been drawn.
  • Keyframing is not playing well with the time slider selector.
  • VisIt issues a warning message about the -o option even though it was not given.
  • VisIt issues a warning message about session files that was unclear to some users.
  • The 3D axis labels for the Surface plot do not change when the plotted variable is changed.
  • The Surface plot does not obey the global rendering settings such as wire-frame and point rendering modes.
  • Mesh opaqueness logic incorrectly applies to point meshes.
  • Pick points are not cleared when the vis window switches from 2D to 3D and vice-versa.
  • The Subset window's default size and shape is too small.
  • The visibility of annotation objects is not preserved when restoring session files that had annotation objects.
  • Setting Lineout operator attributes with the Line tool causes the plot list to briefly show the plot list for the wrong vis window.
  • Single color vector is not reflected in the legend.
  • The Time slider annotation object shrinks by saving and restoring session files.
  • The visit script should understand -h and --help options and print out available command line options appropriately.
  • VisIt needs better designations for ghost zones to allow more kinds of ghost zones.
  • Some operators should not be applied twice.
  • Screen coordinates have been removed as an option from the Query window.
