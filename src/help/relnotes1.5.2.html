VisIt 1.5.2 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.5.2

  • VisIt's Weighted variable sum query now has an option to weight by the revolved volume for 2D datasets that are revolved into 3D.
  • VisIt now intermittently saves the components of your settings that are related to the paths that you've visited so they can be recalled more easily in future VisIt sessions without requiring all settings to be saved.
  • Visit's Silo database reader plugin now supports Silo's Curve objects.
  • VisIt now supports 1-button mice, which are common on MacOS X systems. The default clicking behavior is still mapped to the left mouse button. Right mouse button is simulated by holding down the Alt-key while you click the mouse. Middle mouse button is simulated by holding down Shift+Alt while you click the mouse.
  • VisIt now supports redo view in addition to undo view. Redo view is used to re-apply views that have been undone using undo view.
  • VisIt has a new ijk_gradient expression that can be used for more efficiently creating the gradient for structured meshes. The new expression also does not suffer from any potential sampling errors that can happen when sampling the gradient spatially.
  • VisIt's Resample operator can now resample variables defined on a point mesh onto a rectilinear mesh.
  • VisIt's automatic fullframe detection mode has been made the default so no user intervention is required to stretch long, skinny 2D plots so they use most of the vis window's available space.
  • VisIt's IndexSelect operator can now be used to limit the number of points drawn for point meshes.
  • VisIt now supports ZR meshes in addition to RZ meshes.
  • The visit -movie script now does a better job of using the actual window size stored in a session file when the -geometry option is not specified.
  • The Color table window now ensures that the color table you were editing when you saved your settings is the color table that VisIt displays in the Color table window when you next run VisIt.
  • VisIt now resets the counter used to create file names for families of saved images and geometry.
  • VisIt now uses the Ctrl+P hotkey for printing the active vis window. The Pick window is now activated using Shift+Ctrl+P.
  • VisIt's Curve2D database reader plugin now recognizes Ultra files that end in the ".u" file extension.
  • The visit-install script now permits you to choose the bank from which your parallel jobs will draw their processors. Note that if you have previously saved settings then you may still have to edit your host profiles to change the bank. This option is mostly for new VisIt installations.
  • VisIt's launch script now supports IPV6 host addresses.
  • The visit-install script now allows you to pick from ORNL and NERSC network configurations in addition to the supported LLNL configurations.

Bugs fixed in version 1.5.2

  • Turning off the last light hangs the X-server when VisIt is displayed via X11 on a diskless Linux computer.
  • Saved images were all of the first database time state when saving images off-screen and advancing through time.
  • VisIt's viewer crashes when saving a vis window off-screen when the compute engine has timed out.
  • Uncaught exceptions thrown from a new plot plugin's custom rendering class can cause VisIt's viewer to crash. VisIt's viewer has been improved so that uncaught VisIt exceptions are now all caught at the top level of the viewer's event loop and are much less likely to cause problems.
  • VisIt leaked memory under certain circumstances when compositing z-buffers in the parallel compute engine.
  • VisIt's Variable sum query returns the wrong results if material interface reconstruction has occurred.
  • VisIt's GUI crashes when you manipulate a color table's color control points when you have saved color tables in your .visit directory that have the same names as built-in color tables.
  • Opening files that have enough variables to make automatic variable menu grouping kick in causes VisIt's GUI to crash if there are mesh quality expressions in the variable list.
  • VisIt's compute engine crashes when processing a 2D unstructured CGNS file.
  • VisIt's compute engine experienced an out of memory error when reading Boxlib data files. The problem was caused by a namespace collision in VisIt's source code.
  • Using the log10 scaling option for scalars in the Volume plot when there are negative values causes an uncaught exception in VisIt's viewer, leading VisIt to crash.
  • When VisIt draws Volume plots in multiple windows using the Raycasting method, VisIt can become confused and use the plots from one vis window in another vis window, thus producing incorrect pictures.
  • VisIt's Volume plot does not correctly resample datasets that have ghost zones.
  • Using shadows during scalable rendering for 2D plots causes the compute engine to crash.
  • VisIt's viewer hangs when deleting a plot after doing some picks while in scalable rendering mode.
  • VisIt crashes when opening LEOS data files with many variables.
  • VisIt's CLI hangs when started by the Save movie wizard in the GUI on the Windows platform. The CLI's synchronization code was improved so that it takes far less CPU.
  • The InverseGhostZone operator does not work with SIL selections.
  • The Spatial extents query does not return the original extents when required.
  • If scalable rendering fails when generating a movie in the CLI, the CLI will halt with a Python error instead of fixing the problem and continuing.
  • VisIt's viewer crashes when cancelling a connection to a computer where the mdserver and compute engine will share the same batch job.
  • Numbers are lost from various controls for annotation objects in VisIt's Annotations window if you don't click the Apply button.
  • VisIt's viewer issues a strange warning message about compute engines when scalable rendering mode is set to always.
  • When VisIt is forced to perform material interface reconstruction, there can be an erroneous error message: Material selection must be performed on all domains. Please see a VisIt developer.
  • Shadow strength is not restored correctly from a session file.
  • VisIt 1.5.1 does not honor saved settings with multiple vis windows.
  • VisIt's load balancer becomes confused when looking at multiple datasets where one dataset is stored in a format that can do its own domain decomposition and another format that cannot.
  • The determinant expression does not work for 2D expressions.
  • VisIt's Choose center feature does not work in scalable rendering mode again.
  • Middle mouse interaction with 3D interactors does not work using the SGI executable built on that quad machine.
  • The OnionPeel operator does not work properly when there are multiple "domain" collections in a dataset's SIL.
  • Rubber band lines and boxes for the zoom interactors are not drawn in the MacOS X version of VisIt.
  • The option for determining how VisIt picks a vis window for curves created from queries over time has been clarified in the Global Lineout window.
  • Various debugging error messages are printed to the terminal where VisIt was launched when zooming in on plots using the MacOS X version of VisIt.
  • The screen position control used in the Annotation window to control the position of annotation objects is missing its down arrow.
  • Animation toolbar icons are often out of order.
  • VisIt's BUILD_NOTES contain a few errors.
