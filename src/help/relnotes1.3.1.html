VisIt 1.3.1 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.3.1

  • VisIt's rendering code has been partially rewritten so that it is much faster.
  • The MacOS X version of VisIt now takes as little as 5 seconds to fully start up whereas it used to take over 100 seconds on the same hardware. The MacOS X version of VisIt also has an icon now and the menus no longer switch when you switch between the gui and the viewer. Finally, VisIt has an icon in the dock when VisIt is launched.
  • VisIt has a new Displace operator, which is built by default, that allows you to deform a mesh using a vector variable. The amount of deformation can be increased by bumping up the value for its displacement multiplier.
  • VisIt can now draw point meshes with OpenGL points, which is over 5 times faster than previous methods used to draw points. If you want to make point meshes be drawn as actual OpenGL points, choose the "Point" point type in the Mesh and Pseudocolor plots.
  • You can now set your preferred file format reader when you launch VisIt by providing the -default_format argument on the command line. When you provide this option, you must also give the name of the database reader to use. For example, if you like to use Flash files, you could provide -default_format PDB. This option is useful if you write data to files that do not have the necessary file extensions to help VisIt identify the files.
  • Contouring has been made faster.
  • The Mili support tools for VisIt, as well as Mili database reader support, has been added for MacOS X.
  • The BOV database reader has been enhanced so it can read Ensight BOV files.
  • The Isovolume operator now has support for min/max keywords when setting the min and max used to threshold values.
  • Filename comparison is now case insensitive on the Windows platform.
  • VisIt now issues a warning message when a pick point does not intersect any data.
  • VisIt's 2D clipping code has been improved.
  • The visit -movie script now supports -first, -last arguments.
  • VisIt now reports when there is no data in a visualization window.
  • The scalable rendering threshold can now be set so you have control over the point at which VisIt starts to use scalable rendering instead of hardware accelerated rendering.

Bugs fixed in version 1.3.1

  • The compute engine crashes with an expression that involves a mesh and a vector.
  • Engine ID's in the viewer need to account for closed or crashed compute engines.
  • Sometimes the wrong variables are used when setting the time state via replace when there are multiple time sliders.
  • Pick reports the wrong values.
  • Plotting a nearly degenerate point mesh causes the view to get messed up.
  • Pick does not work on a material-selected plot.
  • VisIt has issues reading some 2D Cosmos++ files.
  • The Pseudocolor plot has the wrong data range.
  • Replacing a file on one host with a file from another host fails and causes the vis window to be unable to display plots.
  • The movie making script, visit -movie does not work on Windows.
  • The MinMax query is reporting -1 for the zone number for Mili files.
  • The Volume query does not work over time when the Isovolume operator is used.
  • The NumZones query does not work with ghost zones.
  • Some FilledBoundary plots use the wrong mixed color when they are drawn with "Clean zones only".
  • Slice by percent results in an empty plot at the boundary.
  • The Isovolume operator does not show the right results when the maximum value is left at 1e+37.
  • The Pseudocolor plot's min/max limits option behaves strangely.
  • When the viewer automatically restarts the engine in order to service a scalable rendering request, the viewer crashes.
  • The viewer can't redraw plots in scalable rendering mode after the compute engine has been killed or if it exits for some reason.
  • The Isovolume operator does not work with 2D databases.
  • Problems with state in the viewer. Exceptions in the database have been re-enabled.
  • Silex is dropping the first character of text strings.
  • When the viewer is set up to use the actual extents, the axes are wrong when in scalable rendering mode.
  • Tensor components can't be accessed with an expression.
  • The Threshold operator does not work with vector variables.
  • The Histogram plot fails on the MacOS X platform.
  • The FilledBoundary plot in VisIt yields different results from MeshTV when "Clean zones only" is used.
  • Silex does not free any memory obtained vis DBGetVar calls.
  • Silex crashes on a 7500-long character array.
  • Parallel options are not grayed out on MacOS X when they should be grayed out.
  • Hostnames get dropped in the file selection menu on MacOS X.
  • Query over time cannot cope with a compute engine crashing.
  • Expressions don't work with queries over time.
  • Unusual behavior with the Isovolume operator when using expressions.
  • Expression centering is incorrect for an expression that combines nodal and zonal variables.
  • The MacOS X version of VisIt takes a long time to start up.
  • VisIt groups files that end in ".visit" when it probably should not.
  • Setting the Tensor plot's scale factor affects the tensor glyph colors.
  • The Slice operator shows the wrong orthogonal axis.
  • The OnionPeel operator attributes window is not initialized after you add a new vis window.
  • Silex does not change its title bar when new files are opened.
  • Transparent plots that use variables with different centerings are not lit properly.
  • The viewer's menu on MacOS X contains hidden items.
  • The visit -movie script does not clean up its MPEG params file.
